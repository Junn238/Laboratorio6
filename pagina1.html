Git
En este apartado veremos más sobre un componente primordial para el uso de Github, "Git"
¿Qué es Git

Git es un sistema de control de versiones distribuido, lo que significa que un clon local del proyecto es un repositorio de control de versiones completo. Estos repositorios locales plenamente funcionales permiten trabajar sin conexión o de forma remota con facilidad. Esto significa que podemos confirmar un trabajo localmente y, a continuación, sincronizar la copia del repositorio con la del servidor.

Aspectos básicos de Git

Al guardar nuestro trabajo Git crea una confirmación, esto es una instantánea de todos los archivos en un momento. Las confirmaciones crean vinculos con otras confirmaciones, formando un gráfico del historial de desarrollo. Las confirmaciones se identifican en Git mediante un hash criptográfico. Es por esto que es imposible realizar cambios, perder la información o dañar los archivos sin que Git lo detecte.

Ramas Archivos y confirmaciones
Git proporciona herramientas para aislar los cambios y volver a combinarlos posteriormente. Las ramas son punteros ligeros que administran esta separación y al crear una rama se puede combinar de nuevo en la rama principal cuando se termine la tarea realizada. Los archivos de Git pueden estar en un estado: modificado, almacenado provisionalmente o confirmado. Cuando se modifica un archivo, este no cambia el historial de desarrollo, se pueden ir combinando pocas modificaciones para finalmente realizar una confirmación y enviar un mensaje diciendo lo que ha cambiado
